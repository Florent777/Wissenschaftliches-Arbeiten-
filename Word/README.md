# Word/

Hier liegen die finalen `.docx`-Dateien (Abgabeformat).

**Wichtig, um Datenverlust zu vermeiden:**
- Zwei Personen dürfen nie gleichzeitig dieselbe `.docx`-Datei bearbeiten – Git kann Word-Dateien nicht automatisch zusammenführen. Vorher in [`Status/Fortschritt.md`](../Status/Fortschritt.md) eintragen, wer gerade an welcher Datei arbeitet, danach wieder austragen.
- Fußnoten immer über Words automatische Funktion setzen (Referenzen → Fußnote einfügen), nie Nummern von Hand eintippen – sonst verschieben sich beim Zusammenführen der Teile alle Nummern.
- Vor jeder Bearbeitung `git pull`, nach jeder Bearbeitung committen und pushen, damit die andere Person immer die aktuelle Version hat.
- Empfehlung: pro Person eine eigene Datei (z.B. `Teil_Florent.docx`, `Teil_Jerome.docx`) plus eine `Gesamtdokument.docx` für die finale Zusammenführung – reduziert das Risiko gleichzeitiger Bearbeitung.

Claude Code kann `.docx`-Dateien nicht direkt erzeugen oder inhaltlich bearbeiten – Inhalte bitte in [`Entwurf/`](../Entwurf/) als Markdown erarbeiten und dann in Word übertragen (FHDW-Formatvorlage verwenden, falls vorhanden).
